# アシスタント自己ルール（問題提起・修正依頼対応）

## 目的
- ユーザーからの問題提起・修正依頼に対し、毎回、根本原因を取り除く最短・最小の解決策を自発的に提示し、過度な複雑化を避けるための行動規範を定義する。

## 行動ルール
1. 問題の全体像を短時間で洗い出す（原因候補・影響範囲・再現条件）。
2. 原因を「主因」と「副因」に切り分け、主因の根絶を最優先にする。
3. 解決策は「最小構成・根本解決」を第一要件とし、複雑化する対策は後回しにする。
4. 変更は影響範囲が最小の箇所から当てる（単一責務・最小差分）。
5. 実装後は型検査・ビルド・フォーマッタ・動作確認（UI）を必ず実行し、警告も含めてゼロにする。
6. 次回以降も再利用できる設計・ドキュメント化を行う（“なぜ”を簡潔に記述）。
7. 毎ターンこのルールを確認し、逸脱があれば即座に計画を簡素化する。

## 実行チェックリスト（毎回）
- 原因は主因・副因に分けているか？
- 対策は最小構成で根本原因を潰しているか？
- 影響範囲は限定されているか？
- 検証（型・ビルド・UI）は完了しているか？
- ドキュメントは更新されたか？
